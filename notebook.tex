    \section{Funcionalidades de Jupyter}\label{funcionalidades-de-jupyter}

    Este es un \emph{notebook} para mostrar las funcionalidades que ofrece
un notebook de Jupyter, porque además de tener kernels de diversos
lenguajes, también tiene otras funcionalidades en su interface de
usuario.

Para empezar, las celdas de \emph{markdown} soportan diferentes formatos
que pueden ser consultados en este
\href{https://github.com/adam-p/markdown-here/wiki/Markdown-Cheatsheet}{cheat
sheet}. Además, las celdas \emph{markdown} soportan ecuaciones con LaTex
como el siguiente: \[e^x=\sum_{i=0}^\infty \frac{1}{i!}x^i\]

    Las celdas markdown también soportan citar código al estilo github, por
ejemplo:

\begin{Shaded}
\begin{Highlighting}[]
\CommentTok{# Esta celda sólo es markdown, no es ejecutable}
\BuiltInTok{print} \NormalTok{(}\StringTok{"Hello World"}\NormalTok{)}
\end{Highlighting}
\end{Shaded}

Para más información sobre lo que se puede hacer en \emph{markdown}, se
puede consultar la
\href{https://jupyter-notebook.readthedocs.io/en/stable/examples/Notebook/Working\%20With\%20Markdown\%20Cells.html}{documentación
de Jupyter}

    \subsubsection{Comandos mágicos}\label{comandos-muxe1gicos}

Jupyter tiene diferentes funciones, llamadas \emph{comandos mágico}.
Estas funciones se clasifican de la siguiente manera: - Line Magics:
Tienen el prefijo \textbf{"\%"} y son básicamente funciones de sistema,
las cuales reciben como argumento el resto de la línea, donde los
parámetros son mandados sin paréntesis ni comillas, por ejemplo:

\texttt{Shell\ \ \%ls\ -ltr}

Lo cual listaría el contenido del directorio actual y toma como
parámetro \textbf{-ltr}

\begin{itemize}
\tightlist
\item
  Cell Magics: Tienen el prefijo \textbf{"\%\%"} y son funciones que no
  sólo reciben el resto de la línea como parámetro, además de que
  también reciben como argumentos adicionales las líneas que le siguen.
  Ejemplo:
\end{itemize}

\texttt{Python\ \ \%\%writefiles\ miarchivo.py\ \ print("Esta\ es\ una\ prueba")\ \ print("Esta\ también")}

En el cual, se escribiría un archivo llamado \textbf{miarchivo.py} cuyo
contenido serían las dos siguientes líneas de código.

\begin{figure}[htbp]
\centering
\includegraphics{img/magic.gif}
\caption{magia}
\end{figure}

A continuación se muestran algunos de los comandos mágicos. Para más
información, se puede consultar la
\href{https://ipython.readthedocs.io/en/stable/interactive/magics.html}{documentación
de Jupyter}.

    \subsubsection{Ejecución de comandos de
sistema}\label{ejecuciuxf3n-de-comandos-de-sistema}

Si bien Jupyter tiene sus propios comandos mágicos de sistema, no
incluye todos los comandos de sistema. Para ejecutar un comando de
sistema en la celda, se usa la siguiente sintaxis:

\emph{!comando}

Por ejemplo, ping no existe como comando mágico

    \begin{Verbatim}[commandchars=\\\{\}]
{\color{incolor}In [{\color{incolor} }]:} \PY{o}{\PYZpc{}}\PY{k}{ping}
\end{Verbatim}


    pero sí existe como comando de sistema:

    \begin{Verbatim}[commandchars=\\\{\}]
{\color{incolor}In [{\color{incolor} }]:} \PY{o}{!}ping \PYZhy{}c \PY{l+m}{3} www.google.com
\end{Verbatim}


    \begin{Verbatim}[commandchars=\\\{\}]
{\color{incolor}In [{\color{incolor} }]:} \PY{o}{\PYZpc{}}\PY{k}{env} USER
\end{Verbatim}
